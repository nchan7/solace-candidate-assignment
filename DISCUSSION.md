## Next Steps

Below are some suggested next steps to continue to improve the front end and back end of the app.

### Front End Improvements:
- Add pagination. Given the database could have hundreds of Advocates, adding pagination would create a more user friendly UI and could offer time savings such that the app does not need to load all Advocates when the page is loaded, increasing the speed and lowering size of the app. 
- Provide an option to sort the table by a column (i.e. alphabetically by first or last name)
- Enhance mobile view of the table and ensure current page is mobile friendly
- Enhance search functionality where a user can select multiple options such as search by location and speciality to enhance a user's experience and allow them to find what they are looking for as soon as possible. 
- Import appropriate font for the site to match Solace's branding (Mollie Glaston)
- Improve the UI for each table row especially the specialty column to turn each specialty into a pill/chip similar to what's shown on the Solace website. An example can be found here: https://www.solace.health/state/pennsylvania
- Ensure Accessibility standards are met using VoiceOver utilities
- Unit Testing to improve code standards

### Back End Improvements: 
- Would need to add additional back end GET endpoints or enhance the query to handle a pagination request
- Add server-side search which would return the Advocates sorted by first or last name
- Format the phone number in the back end with dashes to improve readability
- Conduct error handling
- Include authentication if needed for users or administrators.
